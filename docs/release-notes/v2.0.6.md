## 新功能

### Magisk 模块（全新支持）

- 新增 `Magisk/` 目录，提供面向 Android 的 Magisk / KernelSU / APatch 模块，支持在已 Root 的手机上直接运行呆呆面板，浏览器访问 `http://127.0.0.1:5700` 即可使用。
- 使用现代 `module.prop` 格式，自动按设备架构选择 `arm64` 或 `x86_64` 二进制；要求 Android 8.0（API 26）及以上。
- 模块内置 `ddp` 命令，所有 Docker 场景可用的运维命令（status / backup / task / reset-password 等）在手机上同样可用。
- 模块卡片内提供「运行 / Action」按钮，点击后直接在管理器里查看进程状态、端口监听、最近 60 行 `service.log` 与 `server.log`，以及脚本运行时（Python / Node 等）的自检结果。
- 启动脚本自动把 Termux 的 `bin` 与 `lib` 路径合并到面板进程的 `PATH` / `LD_LIBRARY_PATH`，安装 Termux 并 `pkg install python nodejs git` 后无需额外配置即可跑脚本。用户也可把静态二进制放到 `/data/adb/daidai-panel/bin/` 作为独立方案。
- 模块为纯用户态，不修改 `/system`、不写 `system.prop`、不注册 SELinux 规则，不安装任何 APK；仅写入 `/data/adb/modules/daidai-panel/` 与 `/data/adb/daidai-panel/` 两个目录。
- 卸载默认清空持久化数据目录，做到卸载即还原；可通过 `/data/adb/daidai-panel/.keep_on_uninstall` 标记文件保留数据以便重装后继续使用。
- 支持在管理器内一键更新：新版本发布后，模块卡片直接出现「更新」按钮，点按钮即可完成下载、安装、提示重启的完整流程，数据目录不受影响。

### 容器内账号管理命令

- `ddp` 新增 `reset-password [用户名] <新密码>` 命令，忘记密码时直接在容器或 `adb shell` 中即可重置，无需删除数据库重装；系统中只有一个用户时可省略用户名。
- `ddp` 新增 `reset-username [旧用户名] <新用户名>` 命令，支持重命名用户并校验合法性及重名冲突。
- `ddp` 新增 `list-users` 命令，列出全部用户、角色、最近登录时间，忘记用户名时可直接查询。
- 重置密码 / 用户名时会同步清空对应的 `login_attempts`，避免因历史登录失败记录仍处于锁定状态而无法登录。

## 修复

### 环境变量同名读取错乱

- 修复同名环境变量在脚本中被按 `&` 分割后，同名账号偶尔串号 / 被覆盖的问题。实现统一改为：
  - 查询时显式按 `sort_order DESC, position ASC, created_at ASC, id ASC` 稳定排序；
  - 按 `name` 分组收集同名变量的全部值；
  - 统一通过带转义的拼接规则合并（当值本身含 `&` 时自动升级为 `&&`，与脚本侧约定保持一致）。
- 修复此前调度器中同名变量使用 `map` 覆盖写入导致只有最后一个账号生效的缺陷。

### `ddp reset-login` 全量清除报错

- 修复 `ddp reset-login --all` 以及不带参数时报 `WHERE conditions required`、实际并未清除记录的问题，现在全量清空时会正常执行删除。

### 仪表盘与各页面体验修复

- **仪表盘**：今日执行为 0 时"成功率"卡片不再显示具有误导性的 `0%`，改为占位符 `-`。
- **执行日志**：修复点击"取消"删除对话框后仍继续删除的问题；新增日志下载、复制按钮，便于导出到本地分析。
- **订阅管理**：修复 `row.duration.toFixed(1)` 在 duration 为 `null` 时抛出的空指针错误；GitHub 镜像地址改为可在界面配置，默认 `gh-proxy.com`，并持久化到浏览器本地。
- **用户管理**：补全搜索、分页、2FA 状态列；新增密码强度校验（至少同时包含字母和数字）、修改角色前的二次确认。
- **消息通知**：测试失败现在通过 `ElMessageBox.alert` 展示完整错误详情；保存时对 `headers` 等 JSON 字段和 `smtp_port`/`port` 数字字段做格式校验，避免脏配置落库。
- **定时任务**：批量置顶从串行循环改为并发执行，批量操作更快且失败不阻塞其他项；清理日志对话框修复重复定义与取消误删问题；任务导入新增结构校验。
- **环境变量**：`loadDataDepth` 递归增加 3 层护栏，防止异常情况下的死循环渲染。
- **AI 助手**：默认 Prompt 不再覆盖用户已输入的内容，用户输入会持久化到浏览器本地，刷新页面不丢失。
- **个人中心**：修复用户信息接口异常时的静默失败，现在会给出明确提示。
- **系统设置**：系统配置保存失败时自动重新拉取一次服务端配置，避免本地状态与服务端不同步。
